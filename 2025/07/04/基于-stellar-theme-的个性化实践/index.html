文章页底部添加翻转赞赏码卡片

闲来没事，动手给博客文章页的页脚加了一个小卡片。

具体效果就是当鼠标悬停时，这张卡片翻转，从正面展示二维码，整个过程用 CSS3 实现，不依赖额外的 JavaScript。

实现这个功能的关键，其实就是 CSS3 的 3D 变换。我用的是 transform-style: preserve-3d 配合 transition 来做出旋转动画的。最外层是一个 .custom-flip-card-wrap 容器，里面是 .custom-flip-card，而真正发生旋转的是 .custom-flip-card-inner。这个容器在默认状态下 transform: none，而当鼠标悬停在外层时，它就会被加上一个 transform: rotateY(180deg) 的旋转变换。

正面 .custom-flip-card-front 和背面 .custom-flip-card-back 是两个绝对定位的层，彼此完全重叠。其中背面一开始就被旋转了 180 度，而前面则保持不动。加上 backface-visibility: hidden 的设定后，在旋转过程中就不会看到被遮住的那一面，从而确保内容切换的干净利落。具体代码如下：

浮动标签头像设计

在很长一段时间里，我博客的关于页面总觉得缺少点什么。

直到有一天，我看到​@张洪Heo​的关于页面，模块化设计非常的赏心悦目，所以我便想着也为自己的博客关于页面添加一个头像，周围飘动一些像泡泡一样的标签，围绕头像浮动，很直观的让读者了解博主。

灵感有了，接下来的目标就明确了：实现一个「浮动标签头像」效果，而且只用 HTML + CSS 完成，不依赖 JS。

•	中间是一个圆形头像，带有光晕和阴影
•	标签带有渐变背景、毛玻璃质感，还会轻轻上下浮动
•	悬停头像时会有微微放大动画

• 在手机端隐藏所有浮动标签

动画的核心是一段 @keyframes float，通过 translateY 实现上下浮动，再配合轻微的 rotate 和 opacity 变化，每个标签都呈现出错落有致的飘动节奏，并且通过 transform 控制动画，避免布局抖动，兼顾了性能和美感。此外我还特意使用了 linear-gradient + backdrop-filter 给标签添加渐变和毛玻璃质感。

为了确保这套动画在手机和平板上也能看起来舒服，我也做了完整的响应式调整。屏幕越小，头像和标签也会逐渐缩小，同时像素定位的标签也会贴合边缘，避免重叠。

当然，整个设计的基础是设置一个 relative 的容器，让头像和标签都可以准确地定位。而为了防止主题其他 CSS 干扰，我对关键的 margin 和布局设置了 !important 标记。虽然不推荐频繁使用，但在需要明确样式优先级的时候是必要的。

实现逻辑上也不复杂，只需要一点原生 JS。用 window.addEventListener('scroll') 监听滚动事件，判断一下滚动距离，超过设定值就给按钮加个 .show 的 class，按钮自然就显示出来了。点击按钮后，调用 scrollTo({ top: 0, behavior: 'smooth' }) 就能平滑地回到顶部。而为了避免按钮一开始就莫名其妙地显示在页面上，我在 DOM 加载完成后手动移除了 .show，保证加载初始状态干净利落。

之所以不用 display: none 和 block 来控制显示，是因为这样会让渐隐渐显的动画失效。用了 opacity 和 pointer-events 的组合，一方面可以通过 CSS 的 transition 实现平滑过渡，另一方面还能避免用户在按钮不可见时误点。

整个过程很轻量，不依赖第三方库，也不会对加载速度造成什么影响。按钮尺寸和位置在不同设备上也做了响应式适配，确保无论在哪个终端都能保持美观。添加完成之后，我反复测试了几次，表现很稳定。

在 themes/ 文件夹下看不到 Stellar 主题的文件，无法直接修改模板或 CSS ，这是因为通过 NPM 安装主题的方式会将主题文件存放在项目的 node_modules/ 文件夹中，而不是 themes/ 下。

以 Stellar 为例，实际路径是：node_modules/hexo-theme-stellar/

这种结构的好处是可以轻松升级主题，但缺点是无法直接修改主题文件（否则更新时改动就会被覆盖），因此，我们需要采用 Hexo 官方推荐的方式，用 _config.stellar.yml 来“注入”我们自己的样式，而不是直接改源码。
